# Roadmap

Future work for chezmoi-recipes. None of these are committed to, just ideas worth tracking.

## Provenance comments in compiled-home/

Embed a comment in each file written to `compiled-home/` during overlay, recording where the file came from:

```bash
# chezmoi-recipes: recipes/git/chezmoi/.chezmoiscripts/run_once_install-git.sh
```

This enables smarter guard hooks (see below) and helps with debugging.

## Smart guard hooks

With provenance comments, guard hooks could redirect instead of blocking:

- `chezmoi edit ~/.gitconfig` reads provenance from `compiled-home/dot_gitconfig`, prints "this file comes from recipes/git/chezmoi/dot_gitconfig, opening that instead", and opens the actual source file.
- `chezmoi add ~/.bashrc` (no provenance, new file) copies to `home/dot_bashrc` and re-runs overlay.

Binary files and files without comment syntax would need a sidecar manifest (`compiled-home/.provenance.json`).

## Incremental overlay mode

Skip unchanged files during overlay using checksums or mtimes instead of clearing and rebuilding `compiled-home/` from scratch every time. Reduces latency for large setups.

## `chezmoi-recipes doctor` command

Validate the directory structure: `.chezmoiroot` exists and points at `compiled-home`, `.gitignore` includes `compiled-home/`, `home/` exists, no orphaned state entries.

## Scaffold archetypes

`chezmoi-recipes scaffold` generates a one-size-fits-all skeleton (install script + config file + shellrc.d fragment + chezmoiignore). This doesn't fit all recipes. A shell/base recipe needs loaders (dot_bashrc, dot_zshrc, dot_shellrc), not a config file under `private_dot_config/shell/`.

Support recipe archetypes: `scaffold --type shell`, `scaffold --type tool` (current default), `scaffold --type config-only`, etc. Or at minimum, document that the scaffold is a starting point and users should reshape.

## `init --with-tests` flag

`chezmoi-recipes init` could support a `--with-tests` flag that scaffolds a full test/lint harness: `.tool-versions` (bats, chezmoi), `.devcontainer/` (devcontainer.json + init.sh), `test/e2e/` (test_helper.bash + basics.bats), `.editorconfig`, `.shellcheckrc`, and a `test` target in the Makefile.

The container setup should not assume devcontainers. Some users prefer plain docker/podman. Could offer a choice: `--container=devcontainer`, `--container=dockerfile` (generates a `Dockerfile` + `make test-docker` target), or `--container=none`.

Note: `.editorconfig` and `.shellcheckrc` are now generated by default (no flag needed). This item covers the test infrastructure only.

## Custom linters for recipe conventions

Custom linters runnable via `make check` or `chezmoi-recipes lint`. Examples:

- Every `.tmpl` has a vim modeline
- Every `.sh.tmpl` has the chezmoi delimiter directive
- Every `dot_shellrc.d/*.sh` has `# shellcheck shell=bash`
- Install scripts have an idempotent guard (`command -v` or `[ -d`)

Consider whether this belongs in chezmoi-recipes itself or in the dotfiles repo.

## `.chezmoiexternal` support in recipes

Allow recipes to include `.chezmoiexternal.toml` fragments that get merged into a single `.chezmoiexternal.toml` in `compiled-home/`, similar to how `.chezmoiignore` is merged today.
